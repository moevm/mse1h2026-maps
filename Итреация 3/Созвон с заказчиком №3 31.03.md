# Созвон с заказчиком №3
## Агенда
- **Дата**: 31.03.2026
- **Тема встречи**: Обсуждение результатов 2-й итерации и планов на 3-ю итерацию проекта, демонстрация существующего функционала
- **План встречи**:
  - Сохранение результатов работы с графом. **Итог:** сформировать файл (пока что остановились на PDF), в котором будет изображение графа (интерактивной карты) и текстовая информация по каждому узлу графа.
  - Сценарии использования. **Итог:** добавить 2 сценария использования, один из которых будет показывать работу приложения при выгрузке данных (например, для передачи другому человеку в виде файла), а второй покажет авторизацию пользователя, которая позволит продолжить работу на разных устройствах без потери прогресса. 
  - Нефункциональные требования. **Итог:** выбранная архитектура должна обеспечивать прозрачность распределенного приложения. На текущей итерации (2-й) продумать архитектуру, чтобы на 3-й итерации реализовать уровни прозрачности.
  - Категории пользователей. **Итог:** предложено заменить «Студент» на «Обучающийся».
  - Запрос пользователя имеет несколько значений (например, замок). **Итог:** в таком случае позволить пользователю ограничить предметную область или выдать пользователю список значений для уточнения, после чего уже строить граф.

  - Замена Semantic Scholar. Итог: заменить Semantic Scholar на другой источник из-за недоступности первого.
  - Реализация уровней прозрачности. Итог: реализовать для архитектуры поддержку уровней прозрачности (как можно большего числа).
  - Регистрация пользователей. Итог: добавить возможность регистрироваться в приложении, чтобы в дальнейшем была возможность перехода между устройствами.
  - Пользовательский функционал. Итог: по возможности добавить возможность редактирования, удаления и добавления узла.
